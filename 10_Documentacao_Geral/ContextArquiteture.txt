# CONTEXTO

- **Projeto:** Análise e Modelagem Preditiva de Acidentes em Rodovias Federais Brasileiras (V3).
- **Objetivo Analítico:** Estimar a quantidade de acidentes (valor numérico contínuo) em períodos futuros considerando fatores estruturais e operacionais das rodovias (Forecasting com variáveis exógenas).
- **Fontes de Dados:** Polícia Rodoviária Federal (PRF) e Departamento Nacional de Infraestrutura de Transportes (DNIT).
- **Período de Análise:** 2019 a 2025.

# REGRAS_DE_OURO

- **Definição de Acidente (Sinistro):** Evento de trânsito ocorrido na malha rodoviária federal, altamente assimétrico (maioria de acidentes rotineiros com poucas vítimas; minoria envolvendo veículos de carga/ônibus com alta letalidade), influenciado por variáveis de infraestrutura (tipo de pista, traçado) e operacionais.
- **Definição de Métrica de Negócio:** Capacidade do modelo de reduzir a incerteza estatística para embasar o planejamento operacional e a alocação preventiva de recursos (viaturas, equipes médicas) pelo Poder Público.
- **Métricas de Avaliação Estatística:** 
  - MAE (Erro Médio Absoluto): Para medir a média das diferenças absolutas entre valores observados e previstos.
  - R² (Coeficiente de Determinação): Para medir a proporção da variância.
- **Restrição de Algoritmo:** Utilização EXCLUSIVA do algoritmo *Random Forest Regressor*.
- **Restrição de Pré-processamento:** O modelo DEVE ser treinado SEM normalização ou padronização dos dados (proibido o uso de Min-Max Scaling, StandardScaler ou similares), garantindo a exploração da robustez nativa das árvores de decisão a escalas distintas e a outliers.

# LOGICA_DE_INTEGRACAO

- **Cruzamento de Bases (Join):** A integração entre a base de dados da PRF e a base do DNIT deve ocorrer obrigatoriamente através das chaves compostas `br` e `km`.
- **Referência Espacial:** A variável `km` deve ser tratada como um ponto de referência espacial contínuo. O modelo/pipeline deve utilizar essa coordenada linear para buscar e associar os atributos de infraestrutura correspondentes do DNIT (como `tipo_pista` e `tracado_via`) ao local exato ou trecho mais próximo do sinistro.

## Restrições Metodológicas (Pipeline CRISP-DM)

- **1. Entendimento do Negócio:**
  - Traduzir o problema de negócio em um problema analítico de Regressão.
  - Foco na identificação de interdependências entre atributos (ex: má qualidade da via + curvas acentuadas).

- **2. Entendimento dos Dados:**
  - Realizar Análise Exploratória de Dados (EDA) obrigatória.
  - Executar análise estatística de *Skewness* (assimetria) para mapear a inclinação da distribuição dos dados de acidentes.

- **3. Preparação dos Dados (ETL):**
  - **Extração:** Coleta de dados brutos das bases abertas da PRF e DNIT.
  - **Limpeza:** Tratamento obrigatório de valores ausentes (nulos) e registros duplicados.
  - **Transformação Estrutural:** Executar redução da dimensionalidade e engenharia de atributos.
  - **Engenharia de Atributos (Sazonalidade):** O desenvolvedor/IA deve obrigatoriamente criar colunas derivadas de Sazonalidade a partir da coluna `data_inversa`. Isso inclui a extração e criação de *flags* para 'Fim de semana', 'Feriado' e 'Estação do Ano'.
  - **Features Binárias de Infraestrutura:** Criar colunas booleanas (0 ou 1) a partir das colunas originais `tracado_via` e `tipo_acidente`. Exemplo: `is_curva` (1 se `tracado_via` contiver 'Curva', 0 caso contrário).
  - **Tratamento de Outliers:** Detectar valores extremos estritamente pelo método do Intervalo Interquartil (IQR). Os outliers detectados NÃO devem ser removidos ou tratados como erros, pois a análise de assimetria define que são características reais da base.
  - **Escalonamento:** Estritamente proibida a normalização ou padronização geral para o modelo.

- **4. Modelagem (Random Forest Regressor):**
  - **Amostragem (Bagging):** Gerar amostras *bootstrap* extraindo aleatoriamente os dados.
  - **Proporção de Treino:** Utilizar aproximadamente 63% dos dados originais para o treinamento de cada árvore.
  - **Feature Subspace:** Garantir a seleção aleatória de um subconjunto de variáveis em cada nó para forçar o aprendizado de diferentes padrões de correlação.
  - **Critério de Divisão de Nós:** Utilizar a métrica **MSE (Mean Squared Error)** para avaliar a qualidade das divisões, garantindo a minimização do erro quadrático médio em cada nó, adequado para o problema de regressão.
  - **Validação Interna:** Utilizar os 37% dos dados restantes em cada subconjunto para a estimativa interna *Out-of-Bag (OOB)*.
  - **Validação Externa:** Aplicar técnica de validação cruzada (Cross-Validation) para assegurar a capacidade de generalização.

- **5. Avaliação:**
  - Calcular e registrar obrigatoriamente as métricas MAE e R².

- **6. Implantação:**
  - Exportar os dados processados e as predições do modelo.
  - Carregar os resultados no Power BI para a construção de dashboards interativos focados em segmentações temporais e análises geográficas.
